## Purpose

Structured three-stage workflow for co-authoring documentation — context gathering, refinement and structure, and reader testing — for specs, proposals, PRDs, RFCs, and decision docs.

## Why selected

- **Documentation category anchor** — process-oriented skill that improves any long-form technical writing task
- **Repeatable workflow** — clear stages agents can follow consistently across projects
- **Broad triggers** — applies to specs, proposals, design docs, and internal documentation
- **General-purpose** — not tied to a specific tool or platform
- **Official Anthropic skill** — well documented with practical reader-testing methodology
